# Project structure review

## Keep

```text
server/        Backend API, auth, validation, PostgreSQL / local JSON storage
src/           React + legacy page templates still used by the app
public/images/ Static menu and restaurant images
index.html     Vite entry
package.json   Commands and dependencies
```

## Removed from the delivery zip

```text
node_modules/  Regenerated by npm install or npm ci
dist/          Regenerated by npm run build
.claude/       Local editor / assistant configuration
docs/          Historical review notes, not required for deployment
data/          Runtime JSON fallback; server recreates it locally when DATABASE_URL is empty
FIX_REPORT*.md Historical patch notes
```

## Current deployment direction

For Railway production, add PostgreSQL and set `DATABASE_URL`, `ADMIN_USERNAME`, `ADMIN_PASSWORD`, `TOKEN_SECRET`, and `NODE_ENV=production`.
The backend initializes the schema automatically when `DATABASE_URL` exists.
